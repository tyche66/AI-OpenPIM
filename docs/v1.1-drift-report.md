## V1.1 范围边界

- 本阶段不扩张到多租户、ERP、CRM、完整 V2 或多模态 Agent。
- 本阶段只允许 AI 理解、生成、推荐；价格、库存、成本、供应商敏感事实仍必须通过 Business API 回查。
- 不向源码、文档、Compose、Dockerfile、命令参数或测试报告写入真实外部 AI Key。

## 本轮已落地

- 新增 migration `0008_v11_audit_workflow_ocr` 和 `0009_sample_pilot_product_fields`。
- 新增 `GET /api/v1/audit/operation-logs`，支持 action/module/user/response_code/time/page/size 查询，响应不包含 `request_body`。
- 新增 `POST /api/v1/users/{user_id}/disable`、`POST /api/v1/proposals/{proposal_id}/confirm`、`POST /api/v1/quotations/{quotation_id}/confirm`。
- 将角色更新审计 action 标记为 `role_perm_change`。
- 前端 `/logs` 增加操作审计列表和过滤条件。
- 新增 MinIO 备份/恢复脚本，PostgreSQL 备份/恢复脚本改为必须显式注入密码。

## V1.1 GO/NO-GO

当前判定：PIM 核心 V1.1 GO；纯文本 AI 再次轮换凭据后 GO。

PIM 核心 V1.1 阻塞项已解除：管理员强密码重建通过，稳态性能 P99 小于 200ms，编号生产回归 25/25 PASS。当前范围内的纯文本 AI chat/recommend/polish 真实测试通过，但测试 Key 被终端扫描输出，须再次轮换后方可启用；当前保持 `AI_ADAPTER=none`。多模态、embedding 和向量 RAG 不在当前版本范围。详见 `docs/v1.1-verification.md` 和 `docs/performance-optimization-roadmap.md`。
